# Supplier Sync Token (for RCCL Only) and Api Sync Token — Time Limit

## Overview

This article documents the time-to-live (TTL) behavior for the two sync tokens used in the booking flow: the **Supplier Sync Token** (RCCL/RCG brands only) and the **API Sync Token**. See also *Pay to Supplier: Apply Payment after cruise booking is created* for how these tokens are used in a payment request, and *Release Booking with Supplier* for how the supplier-side lock is released.

## Supplier Sync Token (RCG Brands Only)

- Generated by the **cruise line**, not by Odysseus.
- The TTL is believed to be **10–15 minutes**, but **can run longer in practice** (based on experience) — the exact duration is not guaranteed or fully controlled by Odysseus since the cruise line generates and governs it.
- This token represents an actual **lock on the booking with the cruise line**.
- Calling **Release Booking** can be used to explicitly release this lock with the cruise line rather than waiting for it to expire naturally. See [Release Booking with Supplier](https://supportdesk.odysol.com/portal/en/kb/articles/release-booking-with-supplier).

## Api Sync Token

- Generated by **Odysseus** (not the cruise line).
- Has a fixed **TTL of 30 minutes**.
- **Does not** place a lock on the booking with the cruise line — it is an internal/API-level control only.
- Purpose: Odysseus uses this token to **validate that a "read booking" was performed before a modify is attempted** — i.e., it enforces the read-before-modify sequencing at the API level, rather than representing a supplier-side hold.

## Key Distinction

| | Supplier Sync Token | Api Sync Token |
|---|---|---|
| Generated by | Cruise line | Odysseus |
| Applies to | RCG brands only | All suppliers |
| TTL | ~10–15 minutes (can run longer) | 30 minutes (fixed) |
| Represents a supplier-side lock? | Yes | No |
| Purpose | Holds the booking with the cruise line | Validates read-before-modify sequencing at the API level |
| How to release early | Call Release Booking with Supplier | N/A — expires on its own; not a lock |

## Integration Notes

- These two tokens serve **different purposes** and should not be conflated: expiration of the API Sync Token causes a token expiration error on the API side, while the Supplier Sync Token remaining active means the booking is still locked with the cruise line itself.
- Since the Supplier Sync Token's actual TTL can exceed the expected 10–15 minutes, do not assume it has expired just because that window has passed — if a lock needs to be released deterministically, call Release Booking rather than waiting it out.
- The Supplier Sync Token is scoped to **RCG brands only** (Royal Caribbean Group) — for other suppliers, there is no equivalent supplier-side lock token to manage.
