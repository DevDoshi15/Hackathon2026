# P\&O UK - Mandatory Cruise Line Air Booking Flow

## Overview

P\&O UK enforces a mandatory Cruise Line Air booking flow for sailings sold as **Fly-Cruise** packages. Odysseus has enhanced the booking flow to identify air-mandatory sailings and support the required air booking process across applicable APIs.

## What Changed?

### 1\. Air Mandatory Indicator (PriceIncludes)

#### Overview

P\&O UK does not expose a direct indicator to identify air-mandatory sailings.

Odysseus derives this information by processing the daily air pricing feed provided by P\&O UK and correlating it with sailing data.

#### Business Logic

* Sailings present in the P\&O UK air pricing feed are treated as air-inclusive/air-mandatory.
* Odysseus imports and processes the air pricing feed daily.
* A derived indicator is exposed through the DataCache.

#### DataCache

Consumers should use the **PriceIncludes** element to determine whether a sailing requires mandatory cruise line air.

> \*\*Note:\*\* The `PriceIncludes` value is generated by Odysseus and is \*\*not\*\* supplied by the P\&O UK sailing feed.

\---

### 2\. ListAirGateways

**Endpoint:** `/v2/reservation/cruise/listairgateways`

#### Overview

This existing endpoint now supports P\&O UK.

#### Data Source

Gateway information is imported daily from the cruise line feed because P\&O UK does not provide a real-time API.

#### Request Requirements

To validate gateway availability, call either:

* ListFareCodes
* ListCategories

including the **CruiseLineAir** element.

\---

### 3\. CruiseLineAir

The **CruiseLineAir** element is already supported and should be included in all Cruise Line Air booking requests.

Reference:
https://supportdesk.odysol.com/portal/en/kb/articles/cruise-air-booking-flow

### 4\. ListFareCodes

**Endpoint:** `/v2/reservation/cruise/listfarecodes`

#### Request

Include the **CruiseLineAir** element.

#### Response

* Existing response schema remains unchanged.
* Returns fare codes applicable to Cruise Line Air sailings.

#### Behavior

* CruiseLineAir changes fare code selection logic.
* Identical fare codes between cruise-only and air-inclusive sailings should not be assumed to represent the same inventory or pricing.

#### Sample Error

```json
{
  "isSucceed": false,
  "advisories": \[
    {
      "code": "5124",
      "message": "The transportation is not available for sailing or rate.Dep City on Hold",
      "description": "POC",
      "type": "Error"
    }
  ]
}
```

\---

### 5\. ListCategories

**Endpoint:** `/v2/reservation/cruise/listcategories`

#### Request

Include the **CruiseLineAir** element.

#### Response

* Existing response schema remains unchanged.
* Categories applicable for Cruise Line Air are returned.

#### Behavior

* `AirInclusive = true` indicates category pricing includes air.
* `Price.Id = 1` represents the combined Cruise + Air fare.
* No separate air fare amount is returned by P\&O UK.

Example:

```json
{
  "id": 1,
  "amount": 2729,
  "details": {
    "taxInclusive": false,
    "nccfInclusive": true,
    "airInclusive": true
  },
  "type": "AVGPP"
}
```

Same error scenarios as ListFareCodes apply.

\---

### 6\. ListPrices

**Endpoint:** `/v2/reservation/cruise/listprices`

#### Request

Include the **CruiseLineAir** element.

#### Response

The existing response schema remains unchanged.

When available, the response may include:

```json
{
  "id": 7,
  "code": "AADD",
  "amount": 50,
  "priority": 10,
  "display": true,
  "displayType": 1,
  "rph": 2
}
```

#### Air Pricing Component

* `Price.Id = 7` represents the air pricing component.
* This is existing functionality and not a new enhancement.

#### P\&O UK Specific Behavior

For sailings using the **LON** gateway:

* P\&O UK does **not** return `Price.Id = 7`.
* This behavior is consistent for both **ListPrices** and **ReadFromSupplier**.

\---

### 7\. ReadFromSupplier

**Endpoint:** `/v2/reservation/cruise/readfromsupplier`

#### Request

No changes are required.

#### Response

Additional air reservation information may be returned under:

`CruiseReservation.additionalReservations`

#### Notes

* This is existing functionality.
* Additional reservation data is not limited to air.
* Air reservation details may become available after booking creation.

## Best Practices

* Use `PriceIncludes` to identify mandatory air sailings.
* Always include `CruiseLineAir` for applicable bookings.
* Treat `AirInclusive=true` as the indicator that category pricing includes air.
* Do not assume identical fare codes represent identical pricing contexts.
* Handle API error code **5124** appropriately.

## Compatibility

This enhancement builds upon existing Cruise Line Air functionality and extends support for P\&O UK mandatory air booking without changing existing API schemas.
